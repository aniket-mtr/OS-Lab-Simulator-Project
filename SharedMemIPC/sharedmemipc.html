Inter-Process Communication through Shared Memory

Inter-process communication (IPC) allows cooperating processes to share data and synchronize their operations. There are two major ways to implement IPC. They are Shared Memory and Message Passing.

In IPC through Shared Memory, a memory segment is shared by many processes for communication. This shared memory segment is attached to the address space of the processes, and who wants to communicate using it. Here, we have simulated two processes writing to (the writer) and reading from (the reader) the shared memory segment.



Shared Memory with a Bounded Buffer

In this form of IPC through Shared Memory, we have a limit on the buffer size, or the size of the shared memory segment.

Shared Memory with an Unbounded Buffer

In this form of IPC through Shared Memory, there is no limit on the buffer (or shared memory segment) size.
